# Changelog

All notable changes to this project are documented in this file.

## [0.6.5] - 2026-04-17

### Fixed

- 업데이트 후 "Failed to load Python DLL" 오류를 근본적으로 방지했습니다.
  - 앱 시작 시 이전 실행에서 남은 `_MEI` 임시 폴더를 자동 정리합니다.
  - 현재 실행 중인 프로세스의 폴더는 보호하고, 다른 잔류 폴더만 삭제합니다.

## [0.6.4] - 2026-04-17

### Fixed

- 업데이트 후 기존 모델(EXAONE 3.5)이 설치되어 있는데도 "AI가 준비되지 않았습니다"가 표시되던 문제를 수정했습니다.
  - `check_local_ready`에서도 migration fallback을 인식하도록 보강했습니다.
- "AI 준비 안됨" 안내 메시지를 비개발자도 이해할 수 있도록 구체적으로 변경했습니다.

## [0.6.3] - 2026-04-17

### Fixed

- 업데이트 후 기존 사용자가 "AI가 준비되지 않았습니다" 오류를 겪던 문제를 해결했습니다.
  - EXAONE 4.0 1.2B가 아직 설치되지 않은 경우, 기존 EXAONE 3.5 2.4B로 자동 fallback하여 요약이 중단되지 않습니다.
- 업데이트 설치 시 "Failed to start embedded python interpreter" 오류를 완화했습니다.
  - 업데이트 스크립트의 프로세스 종료 대기를 강화하고, 잔류 `_MEI` 임시 폴더를 정리하도록 개선했습니다.

### Added

- 홈 화면에 모델 마이그레이션 팝업을 추가했습니다.
  - 기존 모델 사용자에게 새 모델 설치를 안내하고, 설치 완료 후 기존 모델 삭제를 선택할 수 있습니다.

## [0.6.2] - 2026-04-17

### Changed

- 로컬 AI 기본 모델을 `EXAONE 3.5 2.4B`에서 `EXAONE 4.0 1.2B`로 교체했습니다.
  - 파라미터 수가 절반(2.4B → 1.2B)이라 더 빠르고, 4.0 세대 업그레이드로 한국어 성능이 향상되었습니다.
  - 64K 컨텍스트와 Reasoning 모드를 지원합니다.
  - 기존 EXAONE 3.5 2.4B는 `기존 모델` 그룹에서 계속 선택·사용할 수 있습니다.

### Added

- 설정 화면에 설치된 로컬 모델 관리 UI를 추가했습니다.
  - 설치된 모델의 용량을 확인하고, 삭제 버튼으로 디스크 공간을 회수할 수 있습니다.
  - 모델 삭제 API(`DELETE /api/local/models/{model_id}`)를 추가했습니다.

## [0.6.1] - 2026-04-17

### Fixed

- `pydantic`와 `pydantic-core`를 호환되는 쌍으로 고정해, Windows 단일 실행 파일에서 시작 시 버전 불일치로 종료되던 문제를 막았습니다.
- CI와 릴리즈 워크플로에 런타임 import 검증을 추가해, 잘못된 `pydantic` 조합이 다시 배포되지 않도록 했습니다.

## [0.6.0] - 2026-04-17

### Added

- 로컬 AI 추천 모델 구성을 새로 정리했습니다.
  - `Gemma 4 E4B`와 `Qwen 3.5 4B`를 추천 모델로 추가하고, 기존 모델은 별도 그룹으로 계속 선택할 수 있습니다.
- 홈 화면에 새 로컬 모델 안내 팝업을 추가했습니다.
  - 로컬 백엔드를 사용하는 경우 한 번만 노출되고, 설정 화면으로 바로 이동할 수 있습니다.

### Changed

- 설정 화면의 로컬 모델 선택 목록을 `추천 모델`과 `기존 모델`으로 구분했습니다.
- 개발자 표시 이름을 `최경찬`으로 바로잡았습니다.

## [0.5.57] - 2026-03-30

### Added

- 동기화 후 변경된 날짜들의 `daily_overview`를 자동으로 다시 생성하도록 연결했습니다.
  - 이제 새 메일을 백업/요약한 뒤, 그 날짜의 개별 이메일 요약을 바탕으로 날짜별 개요도 함께 갱신됩니다.

### Changed

- 홈의 `날짜별 요약` 버튼 의미를 실제 동작에 맞게 정리했습니다.
  - 체크된 날짜 1개를 기준으로 그 날짜의 모든 이메일을 다시 요약하고,
  - 이어서 해당 날짜의 `daily_overview`도 다시 생성합니다.
- `resummarize-day`와 `refresh-overviews`가 같은 날짜별 개요 재생성 helper를 공유하도록 정리했습니다.
  - 날짜 재요약 후 `daily_overview`가 누락되거나 경로별 동작이 달라지는 문제를 줄였습니다.

## [0.5.56] - 2026-03-30

### Added

- 클라우드 멀티모달 요약 경로를 추가했습니다.
  - OpenAI/OpenRouter 호환 payload에 이미지 입력을 함께 보낼 수 있게 확장했습니다.
  - 설정 화면에 `클라우드 멀티모달` on/off 토글을 추가했습니다.
- 설정 화면에서 Vision 지원 클라우드 모델 안내를 직접 볼 수 있도록 개선했습니다.
  - OpenAI/Gemini/Claude preset에 `Vision` 힌트를 표시하고,
  - OpenRouter 모델 목록도 vision 가능 여부를 추론해 현재 선택 모델의 적합성을 안내합니다.

### Fixed

- 클라우드 멀티모달 이미지 선택 규칙을 보강했습니다.
  - 가로 폭이 600px를 넘는 핵심 이미지는 우선 포함하되,
  - 로고/아이콘/작은 배너처럼 요약 품질에 도움이 적은 이미지는 제외합니다.
- 기사형 메일의 상세 요약이 지나치게 얕게 나오는 문제를 완화했습니다.
  - 기사형/soft-wrap 기사형 본문을 더 잘 감지해 broader coverage 경로로 보내고,
  - synthesis 결과가 너무 짧으면 chunk 단위 요약을 다시 활용해 후반부 내용 누락을 줄였습니다.
- fast 로컬 모델 경로의 출력 토큰 예산을 늘려 실제 기사형 메일에서 2~4개 수준으로 과소 요약되던 문제를 완화했습니다.

## [0.5.55] - 2026-03-23

  - 상태 문구에 `nowrap`을 적용해 최신 확인 후 표시가 불안정하게 줄바꿈/이탈하는 현상을 완화했습니다.

## [0.5.54] - 2026-03-23

## [0.5.53] - 2026-03-23

### Fixed

- 전달(FW) 메일 전처리에서 본문이 비어 요약 품질이 급격히 떨어지던 회귀를 수정했습니다.
  - 원인: `prepare_body_for_llm()`가 본문 시작의 `From/Sent/To/Subject` 헤더 블록을 인용 경계로 잘못 판단해 즉시 중단.
  - 대응: 상단 헤더 블록은 건너뛰고 실제 본문을 유지하도록 로직을 변경하고, 전처리 결과가 비면 비헤더 라인으로 복구하는 안전장치를 추가했습니다.
- 전달/인용 텍스트 처리 회귀를 막기 위한 단위 테스트를 추가했습니다.

## [0.5.52] - 2026-03-23

### Fixed

- 일부 환경에서 업데이트 직후 UI가 `ConnectTimeout`으로 보이던 기동 실패를 수정했습니다.
  - 원인: 템플릿 렌더 호출 호환성 이슈로 `/` 라우트가 500(`TypeError: unhashable type: 'dict'`)을 반환하던 문제.
  - 대응: UI 라우트의 `TemplateResponse` 호출을 `request/name/context` 명시 방식으로 통일했습니다.
- 대시보드의 업데이트 "확인" 버튼 UX를 개선했습니다.
  - 최신 버전이면 버튼 옆에 `최신 버전입니다.` 텍스트를 작게 표시하고,
  - 새 버전 발견/오류도 각각 안내 텍스트로 표시합니다.

## [0.5.51] - 2026-03-22

### Fixed

- 요약 파이프라인을 모델 컨텍스트 기반 동적 청크(map) → 통합(reduce) → 문맥 검증(validate) 흐름으로 보강했습니다.
  - 본문과의 토큰 겹침/노이즈 규칙으로 무관한 문장(계절 인사, 안내문, 헤더 에코 등)을 제거합니다.
- 요약 노이즈 필터를 강화해 이름 단독 항목, 타임스탬프, "바로가기 클릭"류 문구가 상세 요약에 섞이는 문제를 완화했습니다.
- 날짜 화면 상단 액션 버튼 영역을 2열 배치(본문/액션)로 재정의해, 버튼 그룹 가로 폭을 명시적으로 확보했습니다.
  - 좁은 화면에서는 1열로 안전하게 전환되며 가로 스크롤 가능한 버튼 툴바를 유지합니다.

## [0.5.50] - 2026-03-22

### Fixed

  - 앱 버전이 갱신되지 않더라도 정적 파일 내용 해시 기반으로 cache-bust 쿼리를 부여합니다.
- 요약 결과에 사람 이름만 단독으로 나오거나, "바로가기 클릭" 같은 안내 문구/타임스탬프가 끼어드는 노이즈를 제거했습니다.

## [0.5.49] - 2026-03-22

## [0.5.48] - 2026-03-22

## [0.5.47] - 2026-03-22

### Fixed

- 요약 포맷 보정 과정에서 "핵심 요약"과 "상세 요약"이 동일 문장으로 중복되는 현상을 방지했습니다.
- LLM 프롬프트를 조정해 1줄 요약/빈약한 요약이 나오지 않도록(불릿 수/중복 금지) 가드레일을 강화했습니다.
- 날짜 화면 우측 상단 액션 버튼이 세로로 쌓이던 문제를 런타임 보정(JS)까지 포함해 근본적으로 방어했습니다.

### Fixed

## [0.5.45] - 2026-03-21

## [0.5.44] - 2026-03-21

## [0.5.43] - 2026-03-21

### Fixed

- 로컬 모델(Qwen/EXAONE 등) 사용 시 장문 메일 chunking 단계에서 LLM 호출 횟수가 과도해 전체 처리 시간이 길어지며 timeout이 발생하던 문제를 완화했습니다.
  - `performance` 티어를 `standard`와 동일한 chunking 가드레일로 처리합니다.
  - 로컬(비 cloud) 환경에서 chunking이 발생한 경우, 추가 합성(synthesis) LLM 호출을 생략해 LLM 호출 수를 줄입니다.

## [0.5.42] - 2026-03-21

### Fixed
